# Free Games Notifier

A Node.js/TypeScript application that monitors gaming platforms for temporarily free games and sends notifications through various channels (Discord, Ntfy, etc.).

## Project Structure

- `src/` - TypeScript source code
  - `app.ts` - Entry point
  - `Manager.ts` - Orchestrates platforms and notifiers
  - `Scheduler.ts` - Manages scheduling of checks
  - `ScheduledGameNotifier.ts` - Core notification logic
  - `configs/` - Configuration loading and factory classes
  - `games/` - Game data model
  - `notifiers/` - Notification channel implementations (Discord, Ntfy)
  - `platforms/` - Platform scrapers/API clients (Steam, Epic, GOG, Mocked)
- `config/default.yaml` - Main user configuration (platforms and notification channels)
- `config/archive.csv` - Tracks already-notified games to avoid duplicates
- `dist/` - Compiled JavaScript output (generated by `npm run build`)

## Tech Stack

- **Language**: TypeScript (compiled to ES2020 modules)
- **Runtime**: Node.js 20
- **Package Manager**: npm
- **Key Dependencies**:
  - `puppeteer` + `jsdom` - Web scraping
  - `node-fetch` - HTTP requests
  - `toad-scheduler` - Task scheduling
  - `config` + `js-yaml` - YAML configuration management
  - `csv-parse` - CSV archive management
  - `chalk` - Colored console output

## Running the App

```bash
npm run build:start   # Build TypeScript and run
npm run build         # Build only
npm start             # Run pre-built app
```

## Configuration

Edit `config/default.yaml` to configure:
- Which platforms to monitor (`Steam`, `Epic`, `GOG`, `Mocked`)
- Which notification channels to use (`Ntfy`, `Discord`)
- Platform-specific settings (check intervals, game types)
- Channel-specific settings (webhook URLs, topics)

## Workflow

- **Start application**: Runs `npm run build:start` as a console workflow
